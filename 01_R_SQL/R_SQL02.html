2-1 스키마(Schema)란 무엇인가?

테이블에 객체가 될 데이터의 구조와 형식을 정의하는 것.
CREATE TABLE table_name(
  컬럼명1   data_type,
  컬럼명2   data_type,
)

[실습]
CREATE TABLE  student  (
  id  tinyint NOT NULL,
  name varchar(4) NOT NULL,
  sex  enum('남자', '여자') NOT NULL,
  address varchar(50) NOT NULL,
  birthday datatime NOT NULL, 
  PRIMARY KEY(id)
);

* 버전이 업그레이드 되면서 `id`와 같이 하지 않고 생략이 가능해짐.

테이블 스키마 조회
DESC `테이블명` 

테이블 리스트 보기
SHOW tables;

테이블 제거 
DROP TABLE `테이블명`

GUI 환경을 많이 쓸 경우, 
CREATE TABLE 직접 쳐서 사용하지 않고, GUI를 이용하는 경우가 많이 있다.

2-2 SQL을 이용한 테이블 생성

sqlQuery(my, "drop table student;");
## [1] "No Data"
R_sql <- sqlQuery(my,"
        CREATE TABLE  `student`  (
          `id`  tinyint NOT NULL,
          `name` varchar(4) NOT NULL,
          `sex`  enum('남자', '여자') NOT NULL,
          `address` varchar(50) NOT NULL,
          `birthday` datetime NOT NULL, 
          PRIMARY KEY(`id`)
        );
")
